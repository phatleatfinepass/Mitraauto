# Vercel Deployment Guide for Mitra Auto PWA

## Overview

This project is now configured for deployment to Vercel with full PWA support including:
- Service Worker (`/sw.js`)
- Web App Manifest (`/manifest.webmanifest`)
- Static icons (`/icons/*`)
- iPhone Home Screen installation
- Web Push Notifications
- SPA routing for all pages

## Prerequisites

1. **Vercel Account**: Sign up at https://vercel.com
2. **Git Repository**: Push this project to GitHub, GitLab, or Bitbucket
3. **Environment Variables**: Prepare your Supabase credentials

## Deployment Steps

### 1. Connect Repository to Vercel

1. Go to https://vercel.com/new
2. Import your Git repository
3. Vercel will auto-detect the Vite configuration

### 2. Configure Build Settings

Vercel should auto-detect these settings (already in `vercel.json`):

```
Framework Preset: Vite
Build Command: npm run build
Output Directory: build
Install Command: npm install
```

### 3. Add Environment Variables

In Vercel Dashboard → Settings → Environment Variables, add:

**Required Supabase Variables:**
- `VITE_SUPABASE_URL` - Your Supabase project URL
- `VITE_SUPABASE_ANON_KEY` - Your Supabase anonymous key
- `VITE_SUPABASE_SERVICE_ROLE_KEY` - Your Supabase service role key (for server functions)

**Payment Variables (if using Paytrail):**
- `VITE_PAYTRAIL_MERCHANT_ID`
- `VITE_PAYTRAIL_MERCHANT_SECRET`
- etc. (see your current environment)

### 4. Deploy

Click **Deploy** in Vercel. The build process will:
1. Install dependencies
2. Build the Vite app to `/build`
3. Serve static files from `/build`
4. Configure PWA routes correctly

### 5. Verify PWA Functionality

After deployment, test:

**On iPhone (Safari):**
1. Visit `https://your-domain.vercel.app/pwa/cms`
2. Tap Share icon → Add to Home Screen
3. Open the installed app
4. Service Worker should register
5. Push notifications should work (if enabled)

**Check in DevTools:**
- Application tab → Service Workers → should show registered
- Application tab → Manifest → should load correctly
- Console → no service worker errors

## Key Configuration Files

### `/vercel.json`

Handles:
- **SPA Routing**: All routes redirect to `index.html`
- **Static Assets**: Directly serve `/sw.js`, `/manifest.webmanifest`, `/icons/*`
- **Headers**: Correct MIME types and caching for PWA files
- **Service Worker Scope**: `Service-Worker-Allowed: /` header

### `/vite.config.ts`

- Build output: `build/` directory
- Public assets: `public/` directory (copied to build root)
- React + Tailwind CSS plugins
- Figma asset resolver for imported assets

## Troubleshooting

### Service Worker Not Registering

Check Vercel deployment logs for:
- `sw.js` being copied to build output
- No 404 errors for `/sw.js`

Verify headers:
```bash
curl -I https://your-domain.vercel.app/sw.js
```

Should return:
```
Content-Type: application/javascript; charset=utf-8
Service-Worker-Allowed: /
```

### Manifest Not Loading

Check:
```bash
curl -I https://your-domain.vercel.app/manifest.webmanifest
```

Should return:
```
Content-Type: application/manifest+json
```

### Icons Not Showing

Verify icon exists:
```bash
curl -I https://your-domain.vercel.app/icons/app-icon.svg
```

Check browser console for 404 errors.

### Routes Not Working (404)

The `vercel.json` rewrites config handles SPA routing. If routes fail:
1. Check `vercel.json` is deployed
2. Verify Vercel Dashboard → Settings → General → Framework Preset is set correctly
3. Redeploy after verifying configuration

## Environment Differences: Figma Make vs Vercel

| Feature | Figma Make | Vercel |
|---------|------------|--------|
| Service Worker | ❌ May not serve correctly | ✅ Full support |
| Manifest | ❌ May not serve correctly | ✅ Full support |
| Static Assets | ⚠️ Limited | ✅ Full support |
| Build Process | ⚠️ Live editing | ✅ Production build |
| Environment Vars | ❌ Not supported | ✅ Full support |
| Custom Headers | ❌ Not configurable | ✅ Full support |
| PWA Install | ❌ Unreliable | ✅ Reliable |
| Push Notifications | ❌ May not work | ✅ Works |

## Recommended Workflow

1. **Development**: Use Figma Make for rapid UI prototyping and design iteration
2. **Testing**: Deploy to Vercel preview for testing PWA features
3. **Production**: Deploy to Vercel production domain for end users

## Deployment Commands

```bash
# Local development
npm install
npm run dev

# Local build test
npm run build
npm run preview

# Deploy to Vercel (via Git)
git push origin main  # Auto-deploys if connected

# Or use Vercel CLI
npm i -g vercel
vercel --prod
```

## Custom Domain Setup

1. Vercel Dashboard → Settings → Domains
2. Add your domain (e.g., `mitraauto.fi`)
3. Update DNS records as instructed
4. Vercel handles SSL automatically
